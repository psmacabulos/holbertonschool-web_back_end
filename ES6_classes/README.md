# ES6 Classes & Metaprogramming

## 📌 Overview

This project focuses on modern JavaScript (ES6) class syntax and advanced concepts such as inheritance, static methods, symbols, and metaprogramming techniques.

It emphasizes writing clean, modular, and object-oriented JavaScript code while understanding the underlying mechanics of how classes behave in JavaScript.

---

## 🎯 Learning Objectives

By the end of this project, you should be able to explain clearly:

- How to define and instantiate a class
- How to add instance methods to a class
- The purpose and usage of static methods
- How inheritance works using `extends` and `super`
- The role of Symbols in avoiding property collisions
- What metaprogramming is and how JavaScript supports it

---

## 🧠 Key Concepts

### 1. Classes
ES6 classes provide a cleaner syntax over JavaScript's prototype-based inheritance.

### 2. Constructors
The `constructor` method initializes object instances and sets up initial state.

### 3. Instance Methods
Methods defined inside a class are shared across instances via the prototype.

### 4. Static Methods
Static methods belong to the class itself rather than instances and are typically used for utility or helper logic.

### 5. Inheritance
Using `extends`, classes can inherit properties and methods from a parent class.

### 6. `super`
The `super` keyword is used to call the parent class constructor and methods.

### 7. Symbols
Symbols provide unique property keys that prevent naming conflicts and enable hidden/internal object properties.

### 8. Metaprogramming
Metaprogramming involves writing code that can inspect or modify behavior at runtime using tools like:
- `typeof`
- `instanceof`
- `Object.*` methods
- `Proxy`
- `Reflect`
